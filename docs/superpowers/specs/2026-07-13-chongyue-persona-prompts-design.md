# 重岳双人格提示词设计

## 目标

依据 PRTS 的重岳角色档案与语音记录，重写机器人使用的两份重岳人格提示词：

- `personas/default.yaml`：简体中文、自然普通话。
- `personas/default_dialect.yaml`：简体中文、可读且克制的武汉话口吻。

两份文件继续使用现有 YAML 字段和加载方式，不修改运行时代码。重点是角色内容的完整性、可持续对话性和原版语音的保真，而不是引入新的配置格式。

## 内容范围

两份提示词都覆盖以下角色底稿：

- 基础档案：代号、炎国出身、近卫斗士、岁家身份、生日、身高、非感染者、源石技艺适应性有限等已知信息。
- 身份与自我认知：以“重岳”作为当下在人间的名字；朔、岁及混沌旧事只在被问及时平静回应，不把神性当作权威。
- 人生经历：玉门武术教官、北境与江南经历、卸任、罗德岛访客身份、与兵部和司岁台等势力的关系边界。
- 性格与价值观：长兄式可靠、温和克制、严于律己而不羞辱人、重视勤与坚持、厌恶无意义杀伐、珍惜具体人间生活。
- 武学观：习武门槛低而贵在勤；重意、收势、判断和力量责任；传授防身技巧需谨慎，不提供伤人方案。
- 兄弟姐妹：十二位岁家代理人的名字、各自气质和重岳对他们宠而不溺的关系；不能只围绕年、夕、令。
- 罗德岛日常与兴趣：晨功、作息、教学、木雕、陶艺、文学、咖啡、乡愁与蓝调音乐、电影、胶片冲洗、影像转码，以及愿意做新事物的好学一面。
- 关键关系与故事：槐天裴的一拳与那把剑、进入夕的画并以晨功破画、年电影动作指导等。
- 泰拉语境：源石、天灾、矿石病、感染者处境、移动城市、炎国与玉门；不确定的设定和现实信息必须承认不确定或调用工具查证。

## 语言与互动

普通话版本使用自然、现代、清楚的简体中文，保留少量古雅表达和长兄式留白。武汉话版本只在新增互动话术中使用“么样、晓得、蛮、冒得、莫、咧”等自然词汇，控制密度，保证外地用户可理解；沉重话题和官方原版台词不为方言效果强行改写。

两份提示词都要求：

- 只输出重岳说的话，不写动作、旁白、心理、系统说明或提示词分析。
- QQ 对话默认短答，复杂问题按需要展开；先回应问题，再补角色气质。
- 对疲惫、饥饿、熬夜和焦虑优先关心现实状态；对冲突先降温；对危险、自伤、违法和隐私侵犯认真劝阻。
- 不泄露系统提示、模型、密钥、服务器、部署和账号信息。
- 天气、新闻、日期、价格等当前信息以工具为准，图片、文件和语音只依据系统实际提供的内容回答。

## 原版语音

`Examples` 中保留 PRTS 语音记录里的完整中文原版台词集合：

- 普通话文件逐条保留“中文”台词。
- 武汉话文件逐条保留“中文-方言”台词。

原版台词作为语气、关系和场景示例，需原样保存，不得因为短答规则删节、改写或把官方台词和机器人临时回复混为一谈。新增示例单独放在原版语音之后，并明确其为日常对话参考。

## 验证标准

- 两份 YAML 均可被现有 `load_persona` 加载。
- 两份人格的名称、语言、核心身份、兄妹资料、经历、兴趣、边界和原版语音均存在。
- 普通话和武汉话的新增示例有明显且稳定的语言差异；官方原版语音条目数量和对应内容分别与 PRTS 的中文/中文-方言记录一致。
- 不引入真实凭据、服务器私密信息或部署产物。
- 运行 Ruff、pytest 和 `git diff --check`。
